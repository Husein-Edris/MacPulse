# MacPulse v1.1 — Design

Date: 2026-06-12
Status: Approved (pending spec review)

## Goal

Make MacPulse lighter on CPU/battery/memory, and turn it from a passive monitor
into a control surface: kill processes like Activity Monitor, scan for large
files, view recent private + public GitHub commits as an authenticated user, and
locate/open backups in Finder with richer status.

## Constraints (unchanged project invariants)

- **Zero third-party dependencies.** Everything is Foundation / AppKit / SwiftUI /
  Darwin syscalls / IOKit.
- **Pure-logic vs I/O is a hard boundary.** Anything with logic worth testing goes
  in a pure file (no networking/filesystem) and is added to the `scripts/test.sh`
  compile list with assertions in `Tests/TestRunner/main.swift`.
- **Subprocesses** only via `Shell.run` with an absolute binary path and array
  args — never a shell string. Prefer direct syscalls where possible.
- **Build** is swiftc-direct (CLT only, Swift 5.8 / SDK 13.3): no `@Observable`,
  no bare regex literals, ViewBuilder 10-child cap (wrap in `Group`).
- `MenuBarRenderer` and other AppKit-only files stay out of the pure test list.

## 1. Optimization — drop process scan when closed + slow on battery

**Problem:** today `AppState.refreshSystem()` runs both the cheap kernel sample
*and* a `/bin/ps` subprocess every 5s, unconditionally, even while the popover is
closed. The `ps` spawn is the dominant idle cost.

**Changes:**

- **Split sampling in `AppState`:**
  - `refreshSystem()` → cheap kernel reads only (CPU/RAM/disk) for the menu bar.
  - `refreshProcesses()` → the `ps` call, separate.
- **Gate process scan on popover visibility.** `RootView` gains
  `.onAppear { state.popoverDidOpen() }` / `.onDisappear { state.popoverDidClose() }`.
  An `isPopoverOpen` flag drives a dedicated process timer that exists only while
  open (refresh on open + every ~5s). **No `ps` subprocess runs while closed.**
- **Battery-aware cadence.** New `PowerSource.onBattery` (IOKit
  `IOPSCopyPowerSourcesInfo` / `IOPSCopyProvidingPowerSourceType`, no deps).
  Menu-bar sample interval: 5s on AC, ~12s on battery. The decision is a pure
  `Fmt.sampleInterval(onBattery:)` (tested). The system timer reschedules when the
  power source changes (checked each tick; reschedule only on change).

**Result:** popover closed + on battery → one cheap kernel read every ~12s, zero
subprocesses. Menu bar stays live.

**Pure/tested:** `Fmt.sampleInterval(onBattery:)`.

## 2. Process control — end any process, with confirmation

**Changes:**

- `ProcessItem` gains `pid: Int32`. `ps` args become
  `["-Aceo", "pid,pcpu,pmem,comm", "-r"]`.
- Extract `ps` output parsing from `SystemMonitor` into a **pure `ProcessParser`**
  (`parse(_ output: String) -> [ProcessItem]`), added to the test list with cases
  (well-formed lines, short/garbage lines ignored, pid/cpu/mem parsing).
- `SystemMonitor.sampleProcesses(top:)` returns more rows (default ~10) so the
  expanded list has content; Overview's glance still shows the top 3.
- New **`ProcessControl.terminate(pid:force:) -> Result`** using a direct
  `kill(pid, force ? SIGKILL : SIGTERM)` syscall (no subprocess). On `EPERM`
  (root-owned process) return a friendly failure: "Couldn't end <name> — it's
  owned by the system." No sudo escalation.
- **UI (Overview):** the top-process area gets a "Show all" disclosure expanding
  into a scrollable list (~10 rows): pid · name · CPU% · MEM%, a CPU/RAM sort
  toggle, and per-row Quit / Force Quit behind a `.confirmationDialog`. After a
  successful kill, refresh the process list. Failures surface inline.

**Pure/tested:** `ProcessParser.parse`.

## 3. Large-file scan — home folder, files over a threshold

**Changes:**

- New **`FileScanner.scanLargeFiles(minBytes:)`** (I/O): walks the home directory
  with `FileManager.default.enumerator(at:includingPropertiesForKeys:options:)`,
  **skipping `~/Library`, hidden directories, and symlinks** (avoids TCC prompts
  and noise), collecting regular files `>= minBytes` (default 100 MB), keeping the
  top ~30 by size. On-demand only, run in `Task.detached(.utility)`.
- The selection (filter `>= min`, sort desc, take N) is a **pure
  `LargeFileRanker.top(_ files:limit:)`** over a lightweight `LargeFile {url,
  sizeBytes}` model, added to the test list.
- **UI (Tips tab):** a "Scan large files" button beside the existing storage-hotspot
  scan. Results render as rows of size · filename · truncated path, each with
  **Reveal in Finder** (`NSWorkspace.shared.activateFileViewerSelecting`).
  Reveal-only — deletion happens in Finder (no destructive button in v1.1).

**Pure/tested:** `LargeFileRanker.top`.

## 4. GitHub admin — reuse `gh` CLI token, recent private + public commits

**Changes:**

- New **`GitHubAuth.token() -> String?`** runs `gh auth token` via `Shell.run`,
  trying `/usr/local/bin/gh` then `/opt/homebrew/bin/gh`. The token is held **in
  memory only** for the session — never written to UserDefaults or any file.
- `GitHubService.fetch` takes the token when available and:
  - sets `Authorization: Bearer <token>`,
  - fetches the **authenticated** events feed `/users/{user}/events` (includes
    private activity) instead of `/events/public`,
  - reads `total_private_repos` (or `owned_private_repos`) from `/user`.
- `GitHubParser` (pure, tested) extends event parsing to extract **recent commits**
  from PushEvents: `{repo, message, sha, date, isPrivate}`. `GitHubSnapshot` gains
  `recentCommits: [GitHubCommit]`, `privateRepos: Int?`, and `authenticated: Bool`.
- **Security boundary:** the on-disk UserDefaults cache (`githubSnapshotCache`)
  stores **only public-safe aggregates** (counts, streak, public events). Private
  repo names and private commit messages live **in memory only** and are never
  persisted. Implementation: encode a redacted snapshot to the cache (drop
  `recentCommits` entries flagged private and the private repo names) while keeping
  the full snapshot in memory.
- **UI:** GitHub tab gains a "Recent commits" list (repo · message · relative time ·
  private/public badge, opens commit URL on click) and an
  "Authenticated via gh ✓ / Public only" indicator; Settings echoes the auth state.

**Pure/tested:** `GitHubParser` commit extraction + private/public flagging + the
cache-redaction helper.

## 5. Backups — locate, open in Finder, richer info

**Changes:**

- New **`BackupLocations`** resolves real paths (no hardcoded email):
  - Google Drive root via glob `~/Library/CloudStorage/GoogleDrive-*/My Drive`,
    then `projects-backup` and `claude-backups` subfolders.
  - SSD `/Volumes/SSK SSD` (present only when mounted).
  - Logs `~/Library/Logs/projects-backup.log`, `~/Library/Logs/claude-backup.log`.
  Each resolved location exposes whether it currently exists.
- **UI (Backups tab):** Reveal-in-Finder buttons for the projects-backup folder,
  the claude-backups folder, and the SSD backup (disabled/grayed when unmounted),
  plus buttons to open the two logs. Reuses/extends the existing `Opener` helper
  with `Opener.reveal(path:)`.
- **More info** from the already-decoded `status.json`: surface fields not shown
  today — `archived` (projects archived today), `ranToday`, `dbDumpsToday`, claude
  `driveCopies`, drill `checked` + `detail`, and a "next run" derived from
  `schedule`.

**Pure/tested:** the "next run from schedule" derivation if non-trivial; path
resolution is I/O (not unit-tested, verified manually).

## UI layout summary

Tabs stay at four: **Overview · GitHub · Tips · Backups** (+ Settings gear).

- **Overview:** glance metrics + top-3 processes, with a "Show all" disclosure
  expanding into the interactive, killable process list (sort toggle, Quit/Force
  Quit with confirm).
- **GitHub:** existing stats + recent-commits list + auth indicator.
- **Tips:** improvements + storage hotspots + new large-file scan.
- **Backups:** richer status + reveal-in-Finder + open-logs.

## Testing

- `make test` must stay green; new pure files get assertion coverage
  (`ProcessParser`, `LargeFileRanker`, `Fmt.sampleInterval`, GitHub commit/redaction
  helpers).
- `make app` must build clean (watch the ViewBuilder 10-child cap in expanded
  views — wrap in `Group`).
- Manual verification: kill a throwaway process; scan large files and Reveal one;
  confirm authenticated commits show private repos; Reveal each backup folder;
  observe no `ps` in `sample`/Instruments while the popover is closed.

## Out of scope (v1.1)

- Deleting files / emptying caches from the app (Reveal-in-Finder only).
- `sudo`-escalated process kills.
- Storing a GitHub PAT in Keychain (using `gh` token instead).
- Notarization / app icon / auto-update.
